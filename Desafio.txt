Descrição do Problema: 

Você precisa criar uma API manipular instâncias da classe ContaBancaria que você implementou no Problema 4. Esta API deve permitir a criação, 
manipulação e consulta de contas bancárias através de endpoints RESTful.

Funcionalidades da API

Criar uma nova conta bancária:

Endpoint: POST /contas

Body:

{
  "numeroConta": number,
  "titular": string,
  "saldoInicial": number
}```
Resposta de Sucesso:

{
  "message": "Conta criada com sucesso.",
  "conta": { /* detalhes da conta criada */ }
}```
Erros:

400: Conta já existe.
Depositar em uma conta bancária:

Endpoint: POST /contas/:numeroConta/depositar

Body:

{
  "valor": number
}```
Resposta de Sucesso:

{
  "message": "Depósito de R$X realizado com sucesso!",
  "conta": { /* detalhes da conta */ }
}```
Erros:

404: Conta não encontrada.
Sacar de uma conta bancária:

Endpoint: POST /contas/:numeroConta/sacar

Body:

{
  "valor": number
}```
Resposta de Sucesso:

{
  "message": "Saque de R$X realizado com sucesso!",
  "conta": { /* detalhes da conta */ }
}```
Erros:

404: Conta não encontrada.
400: Saldo insuficiente ou valor inválido.
Consultar o saldo de uma conta bancária:

Endpoint: GET /contas/:numeroConta/saldo

Resposta de Sucesso:

{
  "message": "O saldo da conta é: R$X",
  "conta": { /* detalhes da conta */ }
}```
Erros:

404: Conta não encontrada.
Requisitos

Utilize minimal API’s para implementar a API.
Mantenha o código organizado e modular.
Inclua tratamentos de erros adequados e responda com códigos HTTP apropriados.
Garanta que a API seja RESTful e siga boas práticas de nomenclatura e estrutura de endpoints.
Dicas

Use expressões regulares para validação de entradas, se necessário.
Utilize middlewares para lidar com tratamento de erros.
